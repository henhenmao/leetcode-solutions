## Organization

Solutions are grouped into folders of ten by problem ID for easier browsing:

- **`001-010` … `291-300`** — solutions grouped by problem number on LeetCode
- **`300+`** — everything past problem 300 lives here (should probably make more groups).
- **`unfinished/`** — problems I haven't fully solved or polished up yet.
- **`tools/`** — little helper scripts I wrote to manage things
- newly added files sit in the root until I run the `organize.py` tool to file them away.

Each file is named with its problem ID first, e.g. `1339_maximum_product_of_splitted_binary_tree.py`.

## Tags

Most problems in the `unfinished/` directory have a suffix tag describing what needs work

| Tag | Meaning |
| --- | --- |
| **`NO`** | **Not Optimal** — solution accepted on the LeetCode website, but it shouldn't have been (bad runtime/space), or it cheats by violating a rule stated in the problem. (e.g. space constraint stated in question)|
| **`UF`** | **Unfinished** — no solution written yet—basically telling myself to solve the problem at some point in time|
| **`NC`** | **Not Commented** — solution works, but lacks comments/explanation of the approach and complexity. |
